Les technos de description

Les microformats

Les microformats

  • Compatible HTML (peu importe la version)
  • Simple à mettre en place
  • Bien documenté : microformats.org

Exemple

Exemple

  • nom : omelette aux champignons
  • temps de préparation : 10 minutes
  • temps de cuisson : 5 minutes
  • ingrédients : 4 oeufs, 5 champignons de paris, sel, poivre
  • ...

Les moins

  • Déprecié vu que les microdonnées sont sorties
  • Cela pollue un peu le code HTML
  • Vocabulaire pauvre (et sans namespace)
  • Identifiant (sujet) faible

Les microdatas

Les microdatas

  • Simple à mettre en place
  • Bien structuré
  • Utilisé par tous les grands moteurs de recherche
  • Largement documenté
  • schema.org

Les attributs

  • itemscope : Définition de l'objet (élément html enfant)
  • itemid : L'identifiant de l'objet
  • itemtype :Définition de l'ontologie (url vers schema.org)
  • itemprop : Le prédicat
  • itemref :Associer une balise non-descendantes à l'objet

Les moins

  • Disponible uniquement pour le HTML5
  • Ce n'est pas (encore) une recommandation W3C (mode draft)
  • Pas compatible avec le RDFa

Le RDF

Resource Description Framework

  • Recommandation du W3C depuis 2004
  • Recense un ensemble de langage & d'outils pour le sémantique

La famille RDF

  • RDF Schema : langage pour définir les ontologies (inclus dans le OWL)
  • RDFa & RDF/XML: langage pour décrire les triplets
  • RDF Query langage : exemple SPARQL

RDF Schema

Ce qu'il faut savoir c'est que schema.org est compatible RDF via schema.rdf.org !

RDF as attribute RDFa

  • Directement dans le HTML ... en XML
  • Compatible uniquement XHTML
  • Utilisation des namespaces
  • Toute propriété peut être un sujet (relation via les urls)
